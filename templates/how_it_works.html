How MedEasy Works

MedEasy uses a custom 5-stage NLP pipeline built entirely from scratch — no external AI APIs for the core analysis.

Pipeline Overview

Stage 1 — MedicalTokenizer

Cleans raw text: fixes PDF space-joins, converts lakh notation, removes range columns, normalizes encoding. Inserts newlines before test names to split PDF-extracted text.

Stage 2 — MedicalNER (Named Entity Recognition)

Regex-based extraction of: patient name, age, gender, date, conditions (14 types), medications (30+ drugs).

Stage 3 — LabExtractor

Master regex finds all (test_name, value, unit) triples. Resolves 40+ aliases. Compares against LAB_RANGES → Normal / High / Low / Critical High / Critical Low. Collects suggestion keys.

Stage 4 — ReportTypeClassifier (TF-IDF + Naive Bayes)

Vectorizes text with custom TF-IDF. Predicts one of 10 report categories using Multinomial Naive Bayes (Laplace smoothing α=0.5). Trained on 74 samples, trains in <1s.

Stage 5 — MedicalSimplifier + Translator

Computes overall status from abnormal ratio. Generates plain-language explanation and doctor advice in 6 languages using template system.

Machine Learning Algorithms

TF-IDF Formula

TF(t,d) = count(t in d) / |d|
IDF(t) = log((1 + N) / (1 + df(t))) + 1
TF-IDF = TF × IDF

Naive Bayes Formula

log P(class|doc) = log P(class) + Σ [ x_i × log P(word_i|class) ]
Laplace: P(word|class) = (count + α) / (total + α × |vocab|) [α = 0.5]

Cosine Similarity (Chatbot)

sim(A, B) = A·B / (||A|| × ||B||) threshold = 0.12
